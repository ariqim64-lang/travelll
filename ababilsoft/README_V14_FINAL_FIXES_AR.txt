تحديث V14 - تعديلات المهن والخدمات والتفويض والترحيل

تم تنفيذ التعديلات المطلوبة على نسخة V13 بدون تغيير التصميم العام:

1) المهن:
- رقم المهنة يتعبأ تلقائياً عند إضافة مهنة جديدة.
- الرقم قابل للتعديل يدوياً قبل الحفظ.
- منع التكرار مستمر بإشعار داخلي من النظام.

2) الخدمات والحالات:
- عند عرض أو تعديل الخدمة تظهر حالات الخدمة معها.
- الحالات قابلة للتعديل والإضافة والحذف في وضع التعديل حتى 10 حالات.
- وضع العرض يبقى قراءة فقط بتأثير مموه.

3) تفويض التأشيرات:
- فلتر الكل / المفوضة / غير المفوضة يعمل تلقائياً من نفس القائمة.
- إضافة تعديل بيانات التفويض، خصوصاً رقم التفويض والمكتب والملاحظات.
- إلغاء التفويض بقي بصلاحية منفصلة وبإشعار تأكيد داخلي.

4) ترحيل المعاملات وإلغاء الترحيل:
- إلغاء الترحيل يعكس قيد المكتب المخلص بقيد يومي موحد.
- يوجد خيار خصم عند إلغاء الترحيل: بدون خصم، خصم على الكل، خصم على الشخص الواحد.
- عند الخصم على الكل يقسم المبلغ بالتساوي على المعاملات المختارة.
- عند الخصم على الشخص الواحد يعتمد نفس المبلغ لكل معاملة.
- عند بدون خصم لا تظهر خانة المبلغ.
- يتم إنشاء قيد واحد يجمع أسماء العملاء والمكتب، ويمكن فتحه من عرض القيد.

5) استحقاقات الموظفين:
- البيان التلقائي أصبح أوضح ويحتوي نوع الاستحقاق، اسم الموظف، التاريخ، المبلغ، والعملة.

6) المورد الرئيسي وباقي الجهات:
- إذا كان السجل بدون حساب محاسبي، يتم إنشاء حساب له تلقائياً عند فتح القائمة أو العرض حتى تظهر أزرار كشف/قبض/صرف/قيد مثل البقية.

الفحوصات:
- PHP lint: سليم.
- JavaScript check: سليم.
- ZIP test: سليم.

ملاحظة مهمة:
إذا كانت قاعدة البيانات موجودة مسبقاً، تأكد من تنفيذ تحديثات ALTER TABLE الموجودة آخر database/install.sql أو افتح النظام مرة بعد الرفع ليطبّق ensure schema إن كانت الاستضافة تدعم ADD COLUMN IF NOT EXISTS.
