# AbabilSoft V47 - مراجعة ثانية برمجية / محاسبية / أمنية

تمت مراجعة النسخة بعد V46 وإصلاح نقاط إضافية ظهرت أثناء الفحص الثابت للكود:

1) إصلاح إنشاء إشعارات النظام الأمنية
- تم إصلاح خطأ عدد placeholders في notify_user_v45 كان يمنع إنشاء إشعارات مثل دخول جهاز جديد ومحاولة الدخول الخاطئة.
- أصبح أي فشل في إنشاء الإشعار يسجل في سجل النشاط بدل أن يختفي بصمت.

2) تحسين حالات الإشعارات
- عند فتح صفحة الإشعارات يتم تحويل حالة الإشعارات المستلمة من "لم يصله" إلى "وصل" تلقائياً للمستخدم الحالي.
- المدير أو صاحب صلاحية إدارة الإشعارات يستطيع رؤية إشعارات كل المستخدمين وفرزها حسب المستخدم.
- المستخدم العادي لا يرى إلا إشعاراته أو الإشعارات العامة.

3) تقوية أمان إجراءات الإشعارات والتقارير
- تم تحويل قراءة الإشعار والموافقة عليه من روابط GET إلى نماذج POST محمية بـ CSRF.
- تم تحويل وضع سجل النشاط كمقروء من رابط GET إلى POST محمي بـ CSRF.
- تم الإبقاء على فحص الملكية: لا يستطيع المستخدم قراءة أو اعتماد إشعار لا يخصه إلا المدير أو صاحب الصلاحية.

4) حماية كلمة مرور مدير الفرع
- عند إنشاء فرع واختيار مدير فرع، إذا تم إدخال كلمة مرور يدوياً أقل من 8 أحرف يتم رفضها.
- إذا تركت كلمة المرور فارغة يتم توليد كلمة مرور تلقائية.

5) فحص فني بعد الإصلاح
- تم فحص ملفات PHP باستخدام php -l.
- تم فحص ملفات JavaScript باستخدام node --check.
- تم فحص عدم وجود دوال مكررة.
- تم فحص أن كل route يشير إلى دالة موجودة.
- تم فحص تطابق عدد علامات ? في استعلامات q() مع عدد المعاملات الممررة قدر الإمكان عبر تحليل ثابت.

ملاحظة:
لم يتم تشغيل MySQL فعلياً داخل هذه البيئة، لذلك يلزم اختبار الاستيراد والتشغيل على XAMPP / MariaDB منفذ 3307 عند المستخدم.
