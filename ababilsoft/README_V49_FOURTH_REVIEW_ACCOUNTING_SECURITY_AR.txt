مراجعة V49 الرابعة - إصلاحات محاسبية وأمنية إضافية

تم في هذه المراجعة التركيز على الأخطاء التي لا تظهر بالفحص السطحي، خصوصاً القيود غير المتوازنة وعزل الفروع.

أهم الإصلاحات:
1) ترحيل المعاملات إلى مكتب مخلص أصبح بقيد مزدوج:
   - مدين: تكلفة مكاتب مخلصة للفرع.
   - دائن: حساب المكتب المخلص.
   ولم يعد يسجل طرفاً واحداً فقط.

2) إلغاء الترحيل أصبح بقيد عكسي متوازن:
   - مدين: حساب المكتب المخلص.
   - دائن: عكس تكلفة مكتب مخلص.
   - وفي حالة الخصم يتم تسجيله في حساب خصومات إلغاء الترحيل.
   - تم منع خصم إلغاء الترحيل بعملة مختلفة عن عملة الترحيل حتى لا يختل توازن العملة.

3) عمولة الوكيل أصبحت بقيد مزدوج:
   - مدين: عمولات وكلاء.
   - دائن: حساب الوكيل.

4) مردود العميل أصبح بقيد مزدوج متوازن بدلاً من طرف واحد فقط، مع معالجة خصم المكتب وخصم المكتب المخلص.

5) إصلاح إنشاء الحسابات التلقائية للكيانات:
   - الموردين والوكلاء والمكاتب والبنوك والصناديق والعملاء والموظفين يتم إنشاء حساباتهم حسب فرع السجل نفسه، وليس حسب الفرع المفتوح وقت تشغيل الإصلاح.
   - يمنع هذا إنشاء حسابات يتيمة أو ربط حساب فرع بسجل فرع آخر.

6) تقوية إنشاء الحسابات اليدوية:
   - منع تكرار رقم الحساب.
   - منع تكرار نفس اسم الحساب داخل نفس الفرع ونفس التصنيف.
   - توليد رقم حساب آمن لا يصطدم بحساب سابق.

7) حماية المسودات:
   - المستخدم العادي يرى مسوداته فقط.
   - المدير أو صاحب صلاحية إدارة المسودات يرى كل المسودات.

الفحوصات المنفذة:
- فحص PHP Syntax على index.php.
- فحص JavaScript Syntax لكل ملفات assets/js.
- فحص وجود الدوال المرتبطة بالـ routes.
- فحص عدم تكرار الدوال.
- فحص ضغط النسخة النهائية.

ملاحظة:
لم يتم تشغيل MySQL فعلياً داخل بيئة الفحص لأن خدمة MySQL غير متاحة هنا. يلزم اختبار الاستيراد والتشغيل على XAMPP / MariaDB منفذ 3307.
