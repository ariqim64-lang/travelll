نسخة V48 - المراجعة الثالثة والتقوية البرمجية والمحاسبية

تمت هذه المراجعة على نسخة V47 بهدف تقليل الأخطاء البرمجية والمحاسبية وتقوية فصل الفروع والصلاحيات.

أهم الإصلاحات:
1. تقوية عزل الفروع في الخلفية، وليس في الواجهة فقط:
   - منع استخدام صندوق أو بنك أو مكتب مخلص أو مورد أو وكيل أو موظف من فرع آخر عند الحفظ.
   - فلترة القوائم المنسدلة حسب الفرع الحالي في الأدلة والسندات والمعاملات والتأشيرات.
   - منع تعديل أو فتح سجلات الأدلة التابعة لفرع آخر.
   - منع تحديث معاملات أو تفويضات أو مردودات من فرع آخر عبر إرسال طلب يدوي.

2. إصلاحات محاسبية:
   - تثبيت قيود شراء مخزون التأشيرات على مورد الفرع الحالي فقط.
   - تثبيت تحديثات مخزون التأشيرات عند التفويض أو الإنزال أو المردود داخل الفرع الحالي.
   - تثبيت إلغاء الترحيل حتى لا يغير ترحيلات أو معاملات خارج الفرع الحالي.
   - عرض ميزان المراجعة والشجرة المحاسبية للحسابات العامة وحسابات الفرع الحالي فقط، لتقليل الخلط بين الفروع.

3. إصلاحات أمنية وبرمجية:
   - منع الوصول إلى لوحة النظام عبر route غير معروف بدون تسجيل دخول.
   - منع تنفيذ routes الخاصة بالحفظ والتعديل والحذف عبر GET.
   - حماية قراءة واعتماد الإشعارات بحيث لا يقرأ المستخدم إشعاراً لا يخصه.
   - حماية حذف المسودات بحيث لا يحذف المستخدم إلا مسودته.
   - تشغيل display_errors=0 وتسجيل الأخطاء بدلاً من عرضها للمستخدم.
   - تقليل تنفيذ ترحيلات schema الثقيلة بتخزين حالة التنفيذ داخل الطلب.

4. إصلاحات الفروع:
   - كود الفرع يظل داخلياً وتلقائياً.
   - العمليات لا تعرض خانة فرع للمستخدم العادي.
   - العملية تتسجل على الفرع الحالي للمستخدم.
   - المدير العام إذا انتقل إلى فرع، تسجل عملياته على ذلك الفرع.
   - عند إنشاء فرع وربط مدير، يتم نقل المدير وحسابه إلى الفرع الجديد لضمان تطابق الصلاحيات والحسابات.

5. فحوصات تم تنفيذها:
   - فحص PHP Syntax لكل ملفات PHP.
   - فحص JavaScript Syntax لكل ملفات JS.
   - فحص routes والدوال الأساسية بشكل ثابت.
   - مراجعة يدوية لأماكن الحفظ والتعديل المتعلقة بالفروع والحسابات.

ملاحظة مهمة:
لم يتم تشغيل MySQL فعلياً داخل بيئة المراجعة الحالية، لذلك يجب تجربة الاستيراد والتشغيل على XAMPP/MariaDB منفذ 3307 للتأكد النهائي من توافق قاعدة البيانات على جهاز العميل.
