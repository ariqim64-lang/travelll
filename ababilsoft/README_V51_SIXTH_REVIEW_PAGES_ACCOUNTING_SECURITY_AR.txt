# AbabilSoft V51 - المراجعة السادسة البرمجية والمحاسبية والأمنية

تمت هذه المراجعة على نسخة V50 مع التركيز على فتح الصفحات، حماية مسارات الحفظ والتعديل، سلامة القيود المحاسبية، فصل الفروع، والإشعارات والنشاط التجاري.

## أهم الإصلاحات

1. حماية جميع مسارات العمليات الحساسة بحيث لا تعمل إلا عبر POST، ومنع تنفيذ الحفظ أو التعديل أو الحذف عبر GET.
2. إضافة حارس صلاحيات مركزي لمسارات الحفظ والتعديل والحذف، حتى لا يعتمد النظام على إخفاء الأزرار من الواجهة فقط.
3. إصلاح محاسبة استحقاقات الموظفين: أصبحت تسجل بقيد مزدوج متوازن بدلاً من طرف واحد.
4. إصلاح مردود العميل عند وجود خصم مكتب حتى لا يتم تضخيم ذمة العميل أو الإيرادات بصورة غير صحيحة.
5. إصلاح عرض الفارق في ميزان المراجعة بحيث لا يتم خلط العملات في فارق عام واحد، بل يظهر الفارق حسب كل عملة.
6. تقوية إنشاء الحسابات التلقائية ومنع اصطدام أرقام الحسابات عند التوليد.
7. إضافة تحقق تشغيلي لحقول الربط المحاسبي المهمة مثل journal_id و supplier_journal_id عند التشغيل.
8. مراجعة إضافية لصفحات الإشعارات والنشاط التجاري ومسارات النظام للتأكد من وجود الدوال والروابط.

## الفحوصات المنفذة

- فحص تركيب جميع ملفات PHP باستخدام php -l.
- فحص ملفات JavaScript باستخدام node --check.
- فحص عدم وجود دوال PHP مكررة.
- فحص أن كل route في القائمة له case في switch.
- فحص أن كل case يستدعي دالة موجودة.
- فحص أن مسارات العمليات الحساسة لها case موجودة ومحروسة.
- فحص ملف الضغط النهائي.

## ملاحظات مهمة

لم تتوفر خدمة MySQL/MariaDB داخل بيئة العمل الحالية، لذلك لم يتم تشغيل اختبار استيراد قاعدة البيانات فعلياً على XAMPP. يجب اختبار النسخة على جهاز التشغيل بمنفذ MySQL 3307 للتأكد من الاستيراد والتشغيل النهائي.

النسخة لا تحذف بيانات العميل ولا تعدل بنية النظام جذرياً، وإنما تضيف حواجز أمان ومحاسبة فوق البنية الحالية مع تقليل التداخل بين الصفحات.
