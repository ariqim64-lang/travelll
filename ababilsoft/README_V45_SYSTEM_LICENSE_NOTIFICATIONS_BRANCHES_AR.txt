تم تنفيذ التعديلات التالية داخل النسخة:

1) صفحة إعدادات النظام
- تحويل صفحة إعدادات السندات إلى صفحة إعدادات النظام.
- إضافة تبويبات داخلية: إعدادات المكتب، السندات، الإشعارات، الترخيص.
- رقم الشركة الافتراضي 778661119 قابل للتغيير من إعدادات المكتب.
- عرض وقت اليمن/صنعاء داخل إعدادات المكتب.

2) الترخيص المحلي
- الترخيص يعمل محلياً بدون إنترنت.
- بيانات الشركة الافتراضية للتفعيل:
  اسم المستخدم: Mohammed
  كلمة المرور: Mohammed773557406@@
- دعم ترخيص يومي/أسبوعي/شهري/سنوي.
- السنوي ينتهي في نهاية السنة الحالية.
- اليومي/الأسبوعي/الشهري لا يسمح أن يتجاوز نهاية السنة الحالية.
- عند انتهاء الترخيص يتم تحويل النظام إلى وضع قراءة فقط ومنع الحفظ والتعديل والحذف والترحيل والطباعة والتصدير عبر حارس التراخيص العام.
- بعد 5 محاولات خاطئة في بيانات الترخيص يتم قفل شاشة الترخيص ولا تفتح إلا ببيانات الشركة الصحيحة.
- إضافة مؤشر مرخص/تحذير/منتهي أسفل النظام بجوار الإصدار.
- إضافة حماية أولية من التلاعب بتاريخ الجهاز عبر حفظ آخر وقت تشغيل.

3) الإشعارات
- إصلاح صفحة الإشعارات وإعادة بنائها بشكل منظم.
- إضافة زر إنشاء إشعار من نفس صفحة الإشعارات.
- اختيار كل المستخدمين افتراضياً، أو إلغاء الكل واختيار مستخدمين محددين.
- دعم إشعار يحتاج موافقة، مع زر موافقة وحفظ حالة الرد.
- حالات الإشعار: لم يصله، وصل، تمت القراءة، تمت الموافقة، تم التجاهل، لم يرد.
- إضافة فرز حسب حالة الإشعار.
- إعدادات إشعارات صفحة الإشعارات: المكان، المدة، اللون/النوع، الصوت، زر تجربة.
- الإعداد الافتراضي لإشعارات صفحة الإشعارات: أعلى اليمين، 5 ثواني.
- إشعارات النظام المباشرة لها مربع داخلي في النظام ولا تستخدم إشعارات المتصفح إلا لحالة مغادرة الصفحة مع بيانات غير محفوظة.

4) الأجهزة ومحاولات الدخول
- تسجيل نوع الجهاز، نظام التشغيل، المتصفح، IP في تقرير الدخول والأجهزة.
- إرسال إشعار أمني للمدير عند الدخول من جهاز جديد.
- إرسال إشعار أمني عند محاولة دخول خاطئة مع بيانات الجهاز.
- إضافة جدول تقرير الدخول والأجهزة داخل صفحة التقارير وسجل النشاط.

5) الفروع
- الفروع مرتبطة محاسبياً ومفعلة من إعدادات الترخيص.
- كود الفرع يتم إنشاؤه تلقائياً ولا يظهر في السندات أو القيود أو عمليات الإدخال.
- المستخدم العادي يعمل داخل فرعه تلقائياً ولا تظهر له خانة فرع.
- المركز الرئيسي/المدير العام فقط يستطيع فتح فرع آخر، وأي عملية ينفذها أثناء وجوده داخل فرع معين تسجل على ذلك الفرع.
- الفروع الأخرى للمستخدم العادي تكون عرض فقط حسب الصلاحية.
- تم إبقاء الحسابات والفصل المحاسبي حسب الفرع في الخلفية بدون تغيير واجهة السندات والقيود.

6) الفحص الذي تم داخل بيئة العمل هنا
- تم فحص تركيب PHP باستخدام php -l بدون أخطاء.
- تم فحص وجود دوال لكل المسارات route cases بدون دوال مفقودة.
- لا يتوفر MySQL داخل هذه البيئة، لذلك لم أستطع تشغيل النظام فعلياً على قاعدة بيانات XAMPP من هنا. يجب استيراد install.sql وتشغيله على XAMPP/MySQL 3307 للاختبار النهائي الفعلي.

ملاحظة مهمة:
تم تنفيذ التعديلات داخل البنية الحالية نفسها، ولم يتم إنشاء نظام منفصل أو أكواد فوق أكواد مستقلة. بعض الاختبارات المحاسبية الفعلية تحتاج تشغيل XAMPP وقاعدة ababilsoft_db على جهازك لأن بيئة العمل هنا لا تحتوي mysql client/server.
