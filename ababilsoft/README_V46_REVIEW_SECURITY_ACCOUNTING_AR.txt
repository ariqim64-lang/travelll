نسخة V46 - مراجعة برمجية ومحاسبية وأمنية

تمت مراجعة النسخة السابقة وإصلاح نقاط مهمة:

1) إصلاحات محاسبية:
- تعديل سعر الريال السعودي إلى اليمني من 140 إلى 150 في PHP و JavaScript وقاعدة التثبيت.
- إضافة الطرف الدائن لإيرادات الخدمة عند إنشاء/تعديل المعاملة، حتى لا تبقى معاملة العميل مدينة فقط بدون إيراد مقابل.
- إيقاف التسوية التلقائية الخطرة للقيود غير المتوازنة عند كل فتح للنظام؛ لأن التسوية الآلية قد تخفي أخطاء محاسبية بدل إظهارها للمراجعة.
- الإبقاء على فصل الفروع محاسبياً في الخلفية بدون إظهار كود الفرع في السندات أو القيود.
- جعل حسابات المستخدمين العاديين تتبع فرع الموظف تلقائياً عند الإنشاء.

2) إصلاحات الفروع:
- كود الفرع يتم إنشاؤه تلقائياً داخلياً.
- إخفاء زر فتح الفرع إلا لمن يملك صلاحية إدارة الفروع.
- منع تعديل عميل تابع لفرع آخر؛ يظهر فقط للعرض حسب الصلاحية.
- عند إنشاء فرع مع تحديد مدير فرع يمكن إنشاء حساب مستخدم لمدير الفرع تلقائياً أو باسم/كلمة مرور يدويين.

3) إصلاحات الإشعارات:
- منع قراءة أو اعتماد إشعار لا يخص المستخدم إلا للمدير أو صاحب صلاحية إدارة الإشعارات.
- التحقق من عنوان ومضمون الإشعار قبل الحفظ.
- ضبط قيم مكان الإشعار ولونه ومدته ضمن خيارات آمنة.
- إشعارات صفحة الإشعارات تظل منفصلة عن إشعارات النظام.

4) إصلاحات الترخيص:
- إصلاح بدء الفترة التجريبية: إذا كانت الحالة trial لكن لا يوجد تاريخ انتهاء، يتم ضبط نهاية التجربة لمدة 30 يوماً.
- بقاء الترخيص محلياً.
- استمرار حماية تاريخ الجهاز وتوقيت اليمن/صنعاء.

5) تقوية الأمان:
- إخفاء أخطاء PHP من الواجهة وتفعيل تسجيل الأخطاء بدلاً من عرضها للمستخدم.
- جعل إعداد debug في config = false.
- إضافة إعدادات حماية للكوكيز: HttpOnly + SameSite=Lax.
- إعادة توليد رقم الجلسة بعد تسجيل الدخول لمنع سرقة الجلسة.
- إزالة اسم المستخدم وكلمة المرور الافتراضية من حقول شاشة الدخول.
- إخفاء السبب الفني التفصيلي عن المستخدم عند تعذر فتح صفحة، مع تسجيله في سجل النشاط.
- إضافة حد أدنى لطول كلمة مرور المستخدمين الجدد 8 أحرف.

6) اختبارات تمت داخل بيئة الفحص:
- فحص PHP syntax لكل ملفات PHP.
- فحص JavaScript syntax لكل ملفات JS عبر node --check.
- فحص عدم وجود دوال مكررة.
- فحص وجود دوال لكل routes الموجودة في switch.
- فحص إزالة القيم القديمة لسعر 140 من الكود.

ملاحظة مهمة:
لم يتم تشغيل MySQL فعلياً داخل هذه البيئة لأن خدمة MySQL غير متاحة هنا، لذلك اختبار الاستيراد والعمليات الفعلية على قاعدة البيانات يجب تنفيذه عندك على XAMPP منفذ 3307.
